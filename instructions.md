## Hugo Academic/Personal Website — Full Build Prompt

**Goal:** Build a complete, elegant, and minimalist Hugo website for an academic/researcher persona following Hugo best practices. Use the **Hugo Academic (HugoBlox)** theme or build a clean custom theme from scratch using Hugo modules, partials, and shortcodes.

---

### General Requirements
- Fully responsive, mobile-first design
- Minimalist aesthetic: clean typography (e.g., Inter or Playfair Display), generous whitespace, muted color palette with one accent color
- Dark/light mode toggle
- SEO optimized (Open Graph, Twitter Cards, structured data/JSON-LD)
- Fast load times: lazy-loaded images, minified assets
- Accessible (WCAG 2.1 AA compliant)
- RSS feeds per section
- `config.toml` / `hugo.yaml` properly structured with params for easy customization
- Multilingual-ready structure (even if single language at launch)

### Automation Pipelines

All three pipelines below run as **GitHub Actions workflows** (`.github/workflows/`) on a daily cron schedule and can also be triggered manually via `workflow_dispatch`. Each pipeline commits its generated/updated files back to the repository and triggers a Hugo rebuild and deploy (Netlify or GitHub Pages).

---

#### Pipeline 1 — Auto Blog Post from Latest YouTube Video

**Trigger:** Daily cron (`0 8 * * *`) or on new video detected

**Steps:**
1. **Fetch latest video** via YouTube Data API v3 using a stored `YOUTUBE_API_KEY` secret. Compare against `data/videos.yaml` to detect if a new video was published since last run.
2. **Download transcript** using `youtube-transcript-api` (Python) or the YouTube captions API. Fall back to video description if no transcript is available.
3. **Generate blog post** by calling the Anthropic Claude API (`claude-sonnet-4-20250514`) with a structured prompt:
   - Input: video title, transcript/description, tags, publish date
   - Output: a full Markdown blog post with frontmatter (`title`, `date`, `tags`, `categories`, `description`, `youtube_id`, `draft: false`), structured sections (TL;DR, Key Takeaways, full write-up, embedded video), and a canonical note linking back to the YouTube video
4. **Save** the generated `.md` file to `content/blog/YYYY-MM-DD-slugified-title.md`
5. **Commit and push** changes; Hugo rebuild is triggered automatically

**Secrets required:** `YOUTUBE_API_KEY`, `ANTHROPIC_API_KEY`, `GH_PAT`

**Config file:** `scripts/video_to_blog.py` + `.github/workflows/video_to_blog.yml`

---

#### Pipeline 2 — Auto-Update Publications List

**Trigger:** Weekly cron (`0 6 * * 1`) or manual dispatch

**Steps:**
1. **Fetch publications** from one or more academic sources in priority order:
   - **Semantic Scholar API** (by author ID) — returns structured metadata
   - **Google Scholar** via `scholarly` Python library (fallback, rate-limited)
   - **arXiv API** (for preprints by author name)
2. **Merge and deduplicate** results against existing `data/publications.yaml` using DOI or arXiv ID as the unique key
3. **Enrich new entries** with:
   - Abstract (if missing)
   - Citation count from Semantic Scholar
   - Open-access PDF link (via Unpaywall API using `UNPAYWALL_EMAIL` param)
   - Auto-assigned tags based on title/abstract keyword matching against a configurable `scripts/topics_map.yaml`
4. **Write updated** `data/publications.yaml`, sorted by year descending
5. **Optionally generate** a companion `.bib` file at `static/publications.bib` for download
6. **Commit and push** only if changes are detected (diff check before commit)

**Secrets required:** `SEMANTIC_SCHOLAR_API_KEY` (optional, increases rate limits), `GH_PAT`

**Config file:** `scripts/update_publications.py` + `.github/workflows/update_publications.yml`

---

#### Pipeline 3 — Auto-Update Videos List

**Trigger:** Daily cron (`0 7 * * *`) or manual dispatch

**Steps:**
1. **Fetch all videos** from the configured YouTube channel using YouTube Data API v3 with pagination (handles channels with 50+ videos)
2. **Fetch playlist metadata** to preserve category/playlist groupings
3. **Enrich each video entry** with: title, description, publish date, duration, view count, like count, thumbnail URL, playlist/category, tags
4. **Merge** with existing `data/videos.yaml`, updating view/like counts for existing entries and appending new ones
5. **Write updated** `data/videos.yaml` sorted by publish date descending
6. **Commit and push** only if changes detected

**Secrets required:** `YOUTUBE_API_KEY`, `GH_PAT`

**Config file:** `scripts/update_videos.py` + `.github/workflows/update_videos.yml`

---

#### Shared Pipeline Infrastructure

- All Python scripts live in `scripts/` with a shared `requirements.txt` (`requests`, `pyyaml`, `anthropic`, `youtube-transcript-api`, `scholarly`, `python-slugify`)
- A `scripts/utils.py` module handles: YAML read/write, slug generation, Git commit/push via subprocess, deduplication logic
- All pipelines log run summaries to a `logs/pipeline_runs.yaml` file (date, items added, items updated, errors)
- A **pipeline status badge** is shown on the `/about` page using the GitHub Actions badge URL, so visitors can see the site is actively maintained
- All pipelines are **idempotent** — safe to re-run without creating duplicates
